## Що змінилось у v2.0

### Архітектурні зміни

1. **Об'єднаний DB_Plc_RouteFsm**
   - Тепер один DB замість двох (State + Work)
   - Всі тести працюють з `DB_Plc_RouteFsm.RoutesFsm[i]`

2. **Механізми залишаються під Owner=ROUTE після DONE**
   - ✅ НОВИЙ ТЕСТ: Перевірка що owner НЕ звільняється при DONE
   - ✅ НОВИЙ ТЕСТ: Перевірка що owner звільняється при ABORTED

3. **Підтримка StartParam/StopParam**
   - ✅ НОВИЙ ТЕСТ: Багатопозиційна засувка (відкрити → закрити)
   - ✅ НОВИЙ ТЕСТ: Автоматичне закриття при ABORT

## Список тестових сценаріїв

### Базові сценарії (1-3)

| № | Сценарій | Що перевіряє | Очікуваний результат |
|---|----------|--------------|---------------------|
| **1** | START → DONE | Успішне виконання маршруту | State=DONE, ResultCode=200, Owner=ROUTE ✅ |
| **2** | START → STOP_OP → ABORTED | Контрольована зупинка оператором | State=ABORTED, ResultCode=301, Owner=NONE ✅ |
| **3** | REJECT_BY_OWNER | Спроба пуску зайнятого механізму | State=REJECTED, ResultCode=202 |

**Примітка:** Тести використовують тільки slot 0, 1 (редлери), які є в поточній конфігурації `DB_Mechs`.

## Як запустити тести

### Крок 1: Підготовка

1. Імпортувати всі файли з папки `Routes_Test/` в TIA Portal
2. Переконатись що симулятори увімкнені:
   ```
   DB_SimConfig.Redler[0].Enable := TRUE
   DB_SimConfig.Redler[1].Enable := TRUE
   DB_SimConfig.Gate[0].Enable := TRUE (для тесту 4-5)
   ```

3. Додати виклик у OB (Cyclic Interrupt 100ms або Main):
   ```scl
   // Симулятори (ПЕРЕД DeviceRunner!)
   "FC_SimRedler"(
       RedlerIdx := 0,
       StartupTime_ms := "DB_SimConfig".Redler[0].StartupTime_ms,
       StopTime_ms := "DB_SimConfig".Redler[0].StopTime_ms,
       SimFault_Breaker := "DB_SimConfig".Redler[0].SimFault_Breaker,
       FaultTime_Breaker_ms := "DB_SimConfig".Redler[0].FaultTime_Breaker_ms,
       SimFault_Overflow := "DB_SimConfig".Redler[0].SimFault_Overflow,
       FaultTime_Overflow_ms := "DB_SimConfig".Redler[0].FaultTime_Overflow_ms,
       ManualReset := "DB_SimConfig".Redler[0].ManualReset
   );
   
   // ... інші симулятори
   
   // Device Runner
   "FC_DeviceRunner"(...);
   
   // Route Supervisor (ПІСЛЯ DeviceRunner!)
   "FC_Route_Supervisor"();
   
   // Тести (в кінці)
   "DB_Test_Routes_Instance"();
   ```

### Крок 2: Запуск конкретного тесту

**Online mode в TIA Portal:**

1. Відкрити `DB_Test_Routes_Config`
2. Встановити:
   ```
   TestEnabled = TRUE
   RouteId = 1
   TestScenario = 1  (номер сценарію 1..8)
   ```

3. Спостерігати за змінними:
   - `TestStep` (поточний крок)
   - `TransitionCount` (лічильник переходів FSM)
   - `TestPassed` / `TestFailed`
   - `FailReason` (якщо Failed=TRUE)

| 0.5s | VALIDATING (1) | 0 | 0 | NONE | NONE |

| 0.5s | RUNNING (3) | 0 | 0 | ROUTE | ROUTE |

**Критична перевірка v2.0:**
- ✅ Після DONE механізми залишаються під `Owner=ROUTE`

## Діагностика проблем

### Watch Table для онлайн-моніторингу

```
// === Конфігурація тесту ===
DB_Test_Routes_Config.TestEnabled
DB_Test_Routes_Config.TestScenario
DB_Test_Routes_Config.TestStep
DB_Test_Routes_Config.TestTimer
DB_Test_Routes_Config.TransitionCount

// === Результати ===
DB_Test_Routes_Config.TestPassed
DB_Test_Routes_Config.TestFailed
DB_Test_Routes_Config.FailReason

// === Статус маршруту ===
DB_PlcToScada_RouteStatus.RoutesSts[1].RS_State
DB_PlcToScada_RouteStatus.RoutesSts[1].RS_ResultCode
DB_PlcToScada_RouteStatus.RoutesSts[1].RS_ActiveStep
DB_PlcToScada_RouteStatus.RoutesSts[1].RS_StepSts

// === Механізми ===
DB_Mechs.Mechs[0].OwnerCur
DB_Mechs.Mechs[0].Status
DB_Mechs.Mechs[0].CmdParam1
DB_Mechs.Mechs[1].OwnerCur
DB_Mechs.Mechs[1].Status

// === Snapshot останнього переходу ===
DB_Test_Routes_Config.Snapshot_State
DB_Test_Routes_Config.Snapshot_ResultCode
DB_Test_Routes_Config.Snapshot_Timestamp
```

---

### Типові проблеми

| Симптом | Можлива причина | Рішення |
|---------|----------------|---------|
| `TransitionCount = 0` | Route Supervisor не викликається | Додати виклик у OB1 |
| `TestFailed = TRUE` | Механізми не звільнені після DONE | ❌ BUG FSM v2.0 |
| `State = REJECTED` | Механізм зайнятий | Перевірити OwnerCur перед тестом |
| `State = ABORTED (304)` | Fault під час виконання | Перевірити симулятор |

## 🎯 Checklist перед запуском

- [ ] Симулятори увімкнені (`DB_SimConfig.*.Enable = TRUE`)
- [ ] Route Supervisor викликається в OB1
- [ ] Симулятори викликаються ПЕРЕД DeviceRunner
- [ ] DB_Test_Routes_Instance створено
- [ ] Watch Table налаштована
- [ ] TestEnabled = FALSE (початковий стан)

## 📝 Примітки

1. **Швидкість виконання:**
   - Кожен тест займає ~10-15 секунд
   - `StepDelayMs = 2000` (затримка між кроками)
   - Можна прискорити через зменшення затримок у `DB_SimConfig`

2. **Повторний запуск:**
   - Встановити `TestScenario = 0` → `TestScenario = N`
   - Або змінити номер сценарію (автоматичний reset)

3. **Відладка:**
   - Використовувати `Snapshot_*` для аналізу переходів
   - `TransitionCount` показує кількість змін стану
   - `FailReason` містить текстовий опис помилки
