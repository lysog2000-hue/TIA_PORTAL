## 1. Огляд

Система маршрутів автоматично запускає та зупиняє механізми елеватора у заданій послідовності.

**Призначення:**
- Запуск лінії: редлер → норія → засувка
- Зупинка лінії у зворотному порядку
- Одночасна робота до 12 незалежних маршрутів
- Захист від конфліктів (хто керує механізмом)
- Аварійна зупинка всього обладнання

**Принципи:**
- **SCADA вирішує** — який маршрут запустити
- **PLC виконує** — детерміновано, по кроках
- **Безпека перша** — аварійна зупинка з будь-якого стану

## 2. Як це працює

### 2.1 Схема роботи

```
SCADA → Mailbox → Supervisor → FSM → Арбітр → Механізми
                ↓
              Status → SCADA
```

### 2.2 Компоненти

| Блок | Функція |
|------|---------|
| **Mailbox** | Команда від SCADA (який маршрут, які кроки) |
| **Supervisor** | Приймає команду, перевіряє, передає у FSM |
| **FSM** | Виконує маршрут по кроках (12 шт. незалежно) |
| **Арбітр** | Дає дозвіл на керування механізмом |
| **Механізми** | Редлери, норії, засувки, вентилятори |

### 2.3 Що таке "маршрут"

**Маршрут** — це список кроків. Кожен крок — це команда одному механізму:

```
Крок 0: Slot 5  → START (редлер 1)
Крок 1: Slot 12 → START (норія 1), чекати RUNNING
Крок 2: Slot 20 → START (засувка 1), чекати IDLE
```

**Виконання:**
1. PLC бере крок 0 → видає CMD_START редлеру
2. Чекає поки редлер стане RUNNING
3. Бере крок 1 → видає CMD_START норії
4. Чекає поки норія стане RUNNING
5. І так далі...

## 3. Команди та статуси

### 3.1 Команди від SCADA (Mailbox)

| Команда | Код | Коли |
|---------|-----|------|
| `RT_CMD_NONE` | 0 | Немає команди |
| `RT_CMD_START` | 1 | Запустити маршрут |
| `RT_CMD_STOP_OP` | 2 | Зупинити маршрут |

### 3.2 Статуси від PLC

| Поле | Тип | Значення |
|------|-----|----------|
| `RS_State` | UINT | Поточний стан (див. нижче) |
| `RS_ResultCode` | UINT | Код результату (помилка/успіх) |
| `RS_ActiveStep` | USINT | Який крок виконується (0..63) |

### 3.3 Стани маршруту (RS_State)

| Код | Назва | Значення |
|-----|-------|----------|
| 0 | `IDLE` | Вільний, чекає команду |
| 1 | `VALIDATING` | Перевірка: чи готові механізми |
| 2 | `STARTING` | Захоплення: хто керує механізмами |
| 4 | `RUNNING` | Виконує кроки |
| 5 | `STOPPING` | Зупиняє механізми |
| 7 | `REJECTED` | Відхилено (не почався) |
| 8 | `ABORTED` | Перервано (механізми зупинені) |

### 3.4 Коди результатів (ResultCode)

| Код | Назва | Коли |
|-----|-------|------|
| 200 | `OK_RUNNING` | Успіх, маршрут виконується |
| 301 | `REJ_BY_CONTRACT` | Помилка у структурі кроків |
| 302 | `REJ_BY_OWNER` | Механізм зайнятий іншим |
| 303 | `REJ_NOT_READY` | Механізм не готовий (аварія/локальний) |
| 401 | `ABRT_BY_OPERATOR` | Зупинено оператором |
| 402 | `ABRT_BY_FAULT` | Аварія механізму |
| 403 | `ABRT_BY_LOCAL` | Хтось перейшов у локальний режим |
| 405 | `ABRT_BY_SAFETY` | Аварійна зупинка |

## 4. Стани маршруту

### 4.1 Діаграма переходів

```
         START
    ┌─────────────┐
    │    IDLE     │
    └──────┬──────┘
           │
           ▼
    ┌─────────────┐
    │ VALIDATING  │ ← Перевірка готовності
    └──────┬──────┘
           │
     ┌─────┴─────┐
     │           │
     ▼           ▼
┌─────────┐ ┌──────────┐
│REJECTED │ │ STARTING │ ← Захоплення ownership
└────┬────┘ └────┬─────┘
     │           │
     │           ▼
     │      ┌─────────┐
     └─────►│ RUNNING │ ← Виконання кроків
            └────┬────┘
                 │
           ┌─────┴─────┐
           │           │
           ▼           ▼
      ┌─────────┐ ┌─────────┐
      │STOPPING │ │ ABORTED │
      └────┬────┘ └────┬────┘
           │           │
           └─────┬─────┘
                 │
                 ▼
            ┌─────────┐
            │  IDLE   │
            └─────────┘
```

### 4.2 Опис станів

| Стан | Що відбувається |
|------|-----------------|
| **IDLE** | Маршрут вільний, чекає команду START |
| **VALIDATING** | PLC перевіряє: механізми існують, не зайняті, готові до роботи |
| **STARTING** | PLC захоплює ownership на всі механізми маршруту |
| **RUNNING** | Виконує кроки по черзі |
| **STOPPING** | Зупиняє механізми у зворотному порядку |
| **REJECTED** | Помилка на етапі VALIDATING (механізми не змінили стан) |
| **ABORTED** | Аварія / STOP_OP / SafetyStop (механізми зупинені) |

### 4.3 REJECTED vs ABORTED

|               | REJECTED               | ABORTED              |
|---------------|------------------------|----------------------|
| **Коли**      | Не почався             | Виконувався          |
| **Механізми** | Не змінились           | Зупинені             |
| **Код**       | 3xx                    | 4xx                  |
| **Що далі**   | Можна стартувати знову | Можна стартувати знову |

**Детальніше про ручне керування:**  
**[`manual_control.md`](manual_control.md)** — як оператору керувати механізмами повз маршрути.

## 5. Безпека та аварії

### 5.1 GlobalSafetyStop (найвищий пріоритет)

**Дія:**
- Миттєва зупинка **всіх** механізмів
- Маршрут → `ABORTED` з кодом `405`
- Звільняє ownership механізмів

### 5.2 LocalManual (локальний режим)

**Коли:** Хтось перемкнув механізм у локальний режим (на шафі)

**Дія:**
- Якщо ще не стартував → `REJECTED` (код `303`)
- Якщо виконується → `STOPPING` (код `403`)

### 5.3 Аварія механізму

**Коли:** FLTCode ≠ 0 (автомат, переповнення, тощо)

**Дія:**
- Якщо ще не стартував → `REJECTED` (код `303`)
- Якщо виконується → `STOPPING` (код `402`)

## 6. Приклад маршруту

### 6.1 Завдання

Запустити лінію: **Редлер 1 → Норія 1**

### 6.2 Кроки у SCADA

```
RouteId = 1
RC_Cmd = RT_CMD_START
RC_StepCount = 2

Крок 1:  // 1-based індексація
  RS_Slot = 5       // Редлер 1
  RS_Action = START
  RS_Wait = RUNNING // Чекати поки запуститься

Крок 2:
  RS_Slot = 12      // Норія 1
  RS_Action = START
  RS_Wait = RUNNING // Чекати поки запуститься
```

### 6.3 Виконання у PLC

```
Цикл 1:
  State = VALIDATING
  → Перевірка: Slot 5 і 12 існують, готові

Цикл 2:
  State = STARTING
  → Захоплення ownership на Slot 5 і 12

Цикл 3:
  State = RUNNING, ActiveStep = 1
  → CMD_START(Slot 5)

Цикл 4-10:
  State = RUNNING, ActiveStep = 1
  → Чекаємо поки Slot 5 стане RUNNING

Цикл 11:
  State = RUNNING, ActiveStep = 2
  → CMD_START(Slot 12)

Цикл 12-20:
  State = RUNNING, ActiveStep = 2
  → Чекаємо поки Slot 12 стане RUNNING

Цикл 21:
  State = RUNNING, ActiveStep = 3 (>= StepCount)
  → ResultCode = 200 (OK_RUNNING)
  → Маршрут виконано, механізми працюють
```

### 6.4 Зупинка маршруту

**SCADA видає:** `RT_CMD_STOP_OP`

```
State = STOPPING
→ Крок 1: CMD_STOP(Slot 12) // Норія
→ Крок 0: CMD_STOP(Slot 5)  // Редлер (зворотний порядок)
→ Чекаємо поки обидва зупиняться
→ CMD_RELEASE_OWNER для обох
→ State = ABORTED, ResultCode = 401
```
